03. Ordinal Mapping

We applied manual dictionary mapping to features possessing an inherent hierarchy (e.g., Quality and Condition) to preserve their statistical significance.

map_dict = {'Ex': 5, 'Gd': 4, 'TA': 3, 'Fa': 2, 'Po': 1, np.nan: 0} for col in ord_cols: dfs[col] = dfs[col].map(map_dict)

Why LabelEncoder?

  • Default encoders sort alphabetically.
  • Custom mapping maintains the logical progression (Excellent > Good > Average).

04. Nominal (One-Hot) Encoding

# Creating dummy variables and avoiding multicollinearity dfs = pd.get_dummies(dfs, drop_first=True)

For non-ranked categories, we utilized get_dummies. By dropping the first column, we successfully prevented the Dummy Variable Trap.

Converted discrete categories into expanding binary vectors (0s and 1s).

05. Multivariate Imputation (KNN)

Instead of relying on simple mean/median, we deployed the KNN Imputer. It intelligently predicts missing values by analyzing the multi-dimensional similarity of neighboring properties.

from sklearn.impute import KNNImputer imputer = KNNImputer(n_neighbors=2) df_final = imputer.fit_transform(df_merged)

Zero Missing Values Achieved

Preserving statistical integrity before modeling.

06. Pipelines & Target Transformation

We integrated MinMaxScaler for feature distance normalization, paired with TransformedTargetRegressor to apply a Log Transform to the skewed SalePrice.

pipe = make_pipeline(MinMaxScaler(), KNeighborsRegressor(n_neighbors=5)) model = TransformedTargetRegressor( regressor=pipe, func=np.log1p, inverse_func=np.expm1 )

Strategic Advantages:

  • Prevents Data Leakage during Cross-Validation.
  • Normalizes Target Distribution for better learning.
  • Squeezes features for distance-based models (KNN/SVR).
